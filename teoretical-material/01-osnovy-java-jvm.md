# Основы работы Java: JVM, JDK, JRE

Java — это компилируемый и интерпретируемый язык одновременно.

### 1. Как выполняется код
1. **Компиляция**: Исходный код (`.java`) компилируется с помощью `javac` в **байт-код** (`.class`).
2. **Выполнение**: Виртуальная машина Java (**JVM**) интерпретирует байт-код или компилирует его «на лету» (JIT-компиляция) в машинный код конкретной ОС.

### 2. Компоненты окружения
*   **JVM (Java Virtual Machine)**: Среда выполнения байт-кода. Обеспечивает кроссплатформенность: «Write Once, Run Anywhere».
*   **JRE (Java Runtime Environment)**: JVM + набор стандартных библиотек. Минимум для *запуска* программ.
*   **JDK (Java Development Kit)**: JRE + инструменты разработчика (`javac`, отладчики, профилировщики). Необходим для *создания* программ.

### 3. Управление памятью
*   **Stack (Стек)**: Хранит примитивы и ссылки на объекты. Каждому потоку выделяется свой стек. Работает по принципу LIFO.
*   **Heap (Куча)**: Хранит сами объекты. Общая для всего приложения.
*   **Garbage Collector (GC)**: Автоматически удаляет объекты из кучи, на которые больше нет ссылок.

### 4. JIT-компиляция (Just-In-Time)
Оптимизирует выполнение, превращая часто используемый байт-код в машинный код во время работы программы, что повышает производительность.
