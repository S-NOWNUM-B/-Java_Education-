# Java Collections Framework (JCF)

Иерархия структур данных для хранения групп объектов.

### 1. List (Списки)
Упорядоченные коллекции, элементы доступны по индексу.
*   **ArrayList**: На базе массива. Быстрый доступ по индексу ($O(1)$), медленная вставка/удаление в середине ($O(n)$).
*   **LinkedList**: Двусвязный список. Быстрая вставка/удаление по краям ($O(1)$), медленный поиск по индексу ($O(n)$).

### 2. Set (Множества)
Коллекции, не содержащие дубликатов.
*   **HashSet**: На базе HashMap. Не гарантирует порядок. Самый быстрый ($O(1)$ для поиска/добавления).
*   **LinkedHashSet**: Сохраняет порядок добавления.
*   **TreeSet**: Хранит элементы в отсортированном порядке (Natural ordering или Custom Comparator).

### 3. Map (Словари / Карты)
Хранение пар «Ключ — Значение».
*   **HashMap**: Ключи уникальны. Порядок не гарантирован.
*   **LinkedHashMap**: Сохраняет порядок добавления.
*   **TreeMap**: Сортирует по ключам.

### 4. Queue (Очереди)
Для хранения элементов, предназначенных для обработки.
*   **PriorityQueue**: Элементы извлекаются согласно приоритету.

---

### Полезные утилиты
*   `Collections.sort(list)` — сортировка.
*   `Collections.reverse(list)` — разворот.
*   `Arrays.asList(...)` — быстрая инициализация списка.
