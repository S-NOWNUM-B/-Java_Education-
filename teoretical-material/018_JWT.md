# JWT (JSON Web Token)

Стандарт для создания токенов доступа (RFC 7519). Используется для аутентификации.

### Структура токена (через точку)
1.  **Header**: Алгоритм и тип токена.
2.  **Payload**: Данные (claims), например `user_id`, `role`, `exp` (срок жизни).
3.  **Signature**: Хэш (Header + Payload + Secret Key). Гарантирует, что данные не меняли.

### Процесс (Stateless)
1.  Клиент шлет логин/пароль.
2.  Сервер проверяет и выдает подписанный JWT.
3.  Клиент шлет JWT в заголовке `Authorization: Bearer <token>` при каждом запросе.
4.  Сервер проверяет подпись токена (без обращения в БД).

### Плюсы
*   Не нужно хранить сессии на сервере (Stateless).
*   Легко масштабируется.
