# Объектно-Ориентированное Программирование (ООП) в Java

ООП базируется на 4 основных принципах.

### 1. Инкапсуляция
Скрытие внутренней реализации класса и предоставление доступа через публичные методы (геттеры/сеттеры).
*   **Механизм**: Модификаторы доступа (`private`, `protected`, `public`, default).
*   **Цель**: Защита данных и контроль состояния объекта.

### 2. Наследование
Позволяет создавать новые классы на основе существующих.
*   **Ключевое слово**: `extends`.
*   **Особенности**: В Java поддерживается только *одиночное* наследование классов, но множественная реализация интерфейсов.

### 3. Полиморфизм
Способность объекта принимать множество форм.
*   **Статический (Перегрузка / Overload)**: Методы с одинаковым именем, но разными параметрами в одном классе.
*   **Динамический (Переопределение / Override)**: Изменение логики метода родителя в дочернем классе.

### 4. Абстракция
Выделение значимых характеристик объекта, исключая несущественные.
*   **Инструменты**:
    *   `abstract class`: Может иметь состояние и реализацию методов. От него нельзя создать экземпляр.
    *   `interface`: Описывает *поведение* (контракт). В современных Java может иметь `default` и `static` методы.

---

### Отношения между объектами
*   **IS-A** (является): Наследование (`Собака` - это `Животное`).
*   **HAS-A** (содержит): Композиция или агрегация (`Машина` имеет `Двигатель`).
